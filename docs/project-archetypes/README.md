# Project Archetypes

These four archetypes preserve the same dashboard composition while updating terminology to Architecture V3.

They are the domain-facing view of the platform: the images show how a Project feels in practice, while the example projects show how that same shape is expressed in YAML packages.

## How To Use These

1. Start with [Architecture Overview](../architecture/README.md).
2. Pick the domain archetype closest to your use case.
3. Open the matching [example project](../examples/README.md).
4. Follow through to the [source packages](../../packages/README.md).

## Notes

- The archetype images are authoritative for dashboard composition and information architecture.
- The examples are authoritative for the corresponding package layout.
- The [posters](../posters/README.md) explain platform mechanics rather than domain dashboards.
