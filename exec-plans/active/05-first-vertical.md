# 05 First Vertical

## Objective

Prove the platform abstraction by expressing a decision-oriented experience as configuration rather than separate architecture.

## Why It Matters

If a decision workspace cannot fit the shared model cleanly, the platform abstraction is likely wrong or incomplete.

This is the equivalent of the original first-vertical day: return to the initial use case only after the platform model exists, and test whether the vertical can be expressed as configuration.

## Scope

- workspace type
- work item type
- primary output type
- agent roles
- actions

## Deliverables

- decision workspace vertical doc
- mapping from decision UI to platform objects
- evidence that the first vertical does not require a separate foundational object model

## Dependencies

- domain model
- output model
- workspace model
- agent model
- skill model

## Open Questions

- what is unique to decision work?
- what is only configuration?
- what approval patterns should be first-class?

## Completion Criteria

- decision work fits the shared platform model
- no decision-specific foundational object is required
